# 煤价预测系统统一技术基线

## 1. 系统目标

本系统用于对煤炭价格进行多尺度预测，输出日度市场价、日度合同价、月度市场价与年度市场价，并提供回测与可视化接口。

## 2. 唯一运行根与入口

- 运行根目录：`core/`
- 训练入口：`core/train.py`
- 服务入口：`core/app.py`
- 训练编排主类：`core/src/pipeline.py` 中 `CoalResearchPipeline`

## 3. 主链路基线（唯一口径）

### 3.1 数据链路

1. `core/src/ingestion.py` 采集并落盘 `data/raw/*.csv`。
2. `core/src/pipeline.py::_build_research_dataset` 统一构建训练数据：
   - 结构化数据 + NLP 指数 + 天气聚合按 `date` 对齐。
   - 使用 `core/src/data_contract.py` 校验并输出质量报告。
3. 模型训练只使用 `data/curated/daily.csv`（由 pipeline 统一生成/复用）。

### 3.2 特征与建模链路

1. `core/src/features.py::build_feature_library` 生成时序与交叉特征。
2. `core/src/features.py::select_core_features_xgboost` 在训练段完成特征筛选并冻结列集合。
3. `core/src/models.py` 多尺度建模：
   - 日度：LSTM + Transformer
   - 月度：LightGBM
   - 年度：SVR + Ridge 融合
   - 合同映射：`ContractPriceMapper`

### 3.3 评估与服务链路

1. 在线留出评估：`reports/online_holdout_metrics.json`
2. 滚动回测：`reports/rolling_backtest_folds.csv` + `reports/rolling_backtest_summary.json`
3. 服务侧预测：`core/app.py::predict_next`，与训练侧共享稳定化规则。

## 4. 配置基线

- 模型参数配置目录：`core/configs/model/*.yaml`
- 应用配置目录：`core/configs/app/*.yaml`
- 数据契约文件：`core/config/data_contract.json`

说明：训练/服务代码通过统一配置加载模块读取上述 YAML（若缺失则回退到代码默认值）。

## 5. 产物基线

- 模型产物：`core/models/`
- 报告产物：`core/reports/`
- 训练缓存：`core/reports/cache/`

## 6. 工程约束

- 训练、回测、服务共享同一预测后处理规则，避免离线/在线口径漂移。
- 不允许服务端任意路径读取用户传入文件；仅允许白名单目录。
- 特征筛选必须在训练段拟合，不得使用测试段信息。
